# 题目分析 | Problem Analysis

## 解题方法论

面对电赛题目，推荐按以下步骤拆解分析：

```
1. 通读题目，圈出所有指标要求
       ↓
2. 划分子系统（电源、采集、处理、执行、显示）
       ↓
3. 选定核心芯片 / 模块（查手册、看方案）
       ↓
4. 画系统框图，确定各子系统接口
       ↓
5. 分工并行开发（硬件 + 软件同步）
       ↓
6. 分模块验证后进行系统联调
       ↓
7. 撰写设计报告
```

## 视觉类题目分析框架

### 系统设计要素

| 要素 | 关键问题 | 典型选择 |
|------|---------|---------|
| 视觉平台 | 算力需求 vs 功耗/尺寸 | 树莓派（重视觉）/ OpenMV（轻量） |
| 摄像头 | 帧率、分辨率、安装方式 | 正射 / 斜射，固定 / 云台 |
| 目标特征 | 颜色、形状、纹理、大小 | 决定识别算法选型 |
| 坐标映射 | 像素坐标 → 物理坐标精度 | 标定板 + 透视变换 |
| 响应时间 | 从检测到执行的延迟要求 | 决定帧率和算法复杂度 |

## 参考题目：运动目标识别与追踪小车

**题目要点（虚拟）**：
- 小车在白色地面上运动，摄像头俯视拍摄
- 识别红色圆形目标，驱动小车到达目标位置
- 目标可能移动，要求实时追踪

**方案设计**：

```
摄像头（俯视，640x480）
    │
    ├── 颜色识别（HSV 红色分割）
    ├── 轮廓提取（找最大圆形区域）
    └── 计算目标中心像素坐标 (cx, cy)
         │
         ├── 误差计算：dx = cx - 320, dy = cy - 240
         └── PID 控制器
              ├── 左右误差 dx → 转向 PWM
              └── 前后误差 dy → 速度 PWM
                   │
                   └── 电机驱动（L298N / DRV8833）→ 小车运动
```

## 参考题目：音频信号分析仪

**题目要点（虚拟）**：
- 输入音频信号，实时显示频谱
- 测量信噪比、THD（总谐波失真）

**方案设计**：
- 前端：低噪声运放采集（增益可调，LM358 / TL072）
- 数字化：STM32 ADC（12bit，采样率 ≥ 44.1kHz）
- 处理：FFT（快速傅里叶变换）
- 显示：OLED / LCD 屏幕
